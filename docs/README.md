# @voxgig/util documentation

This documentation is split into four sections. Each one has a different job, so
reach for the one that matches what you need right now.

## [Getting started](getting-started.md)

A hands-on, start-to-finish walkthrough. Follow it in order and you will have
used most of the library by the end. Best if you are new to `@voxgig/util` and
want to learn it by doing.

## [How-to guides](how-to-guides.md)

Short, focused recipes for specific tasks — "how do I sort a collection?",
"how do I read a deeply nested value?". Best when you already know the library
and have a particular goal.

## API reference — [TypeScript](api-typescript.md) · [Go](api-go.md)

The precise, exhaustive description of every export: signatures, parameters,
return values, edge cases, and short examples. Best when you need to confirm
exactly what a function does.

## [How it works](explanation.md)

The background and design discussion: why there are two implementations, how
parity is maintained, the deliberate differences between the TypeScript and Go
versions, and the reasoning behind the trickier functions. Best when you want
to understand the library rather than just use it.
